## 1. 产品概述

纯前端本地积分打卡工具，支持任务管理、打卡积分、积分统计、积分兑换四大核心功能，数据保存在本地（LocalStorage），无需登录、开箱即用。面向需要每日自律打卡并量化激励的个人用户。

## 2. 核心功能

### 2.1 用户角色

| 角色 | 注册方式 | 核心权限 |
|------|----------|----------|
| 单用户 | 无需注册 | 使用全部功能 |

### 2.2 功能模块

1. **首页**：今日任务面板 + 快捷打卡 + 积分概览（本日/昨日/累计/未完成积分）
2. **任务管理**：添加/编辑/删除任务，区分固定任务与临时任务
3. **历史记录**：按日期筛选查看打卡记录，支持复制导出
4. **数据统计**：日/周积分统计，柱状图可视化，核心数据面板
5. **积分兑换**：折扣兑换 + 兑换记录管理

### 2.3 页面详情

| 页面名称 | 模块名称 | 功能描述 |
|----------|----------|----------|
| 首页 | 积分概览面板 | 展示本日获得、昨日获得、累计总积分、今日未完成积分 |
| 首页 | 今日任务列表 | 分类展示未完成/已完成任务，支持一键打卡 |
| 首页 | 快捷添加任务 | 底部快捷入口，可快速添加临时任务 |
| 任务管理 | 任务列表 | 展示所有固定/临时任务，支持编辑积分/名称、删除 |
| 任务管理 | 添加任务表单 | 输入任务名称、积分值、任务类型（固定/临时） |
| 历史记录 | 日期筛选 | 支持今日/昨日/近7天/自定义日期筛选 |
| 历史记录 | 记录列表 | 每条记录含任务名称、类型、积分、打卡时间、完成状态 |
| 历史记录 | 导出功能 | 一键复制当日/本周所有记录为纯文本 |
| 数据统计 | 核心数据面板 | 本日获得、昨日获得、累计总积分、未完成积分 |
| 数据统计 | 周统计图表 | 周一至周日每日积分柱状图 |
| 数据统计 | 周汇总数据 | 周总积分、日均积分、最高单日积分 |
| 积分兑换 | 兑换档位 | 100积分=95折、300积分=9折、500积分=8折 |
| 积分兑换 | 兑换记录 | 按时间倒序展示，支持清空（二次确认） |

## 3. 核心流程

### 3.1 每日打卡流程

用户打开应用 → 查看今日未完成任务 → 点击打卡（默认当前时间，可自定义）→ 积分实时累加 → 统计数据自动更新 → 全部完成时触发庆祝动效

```mermaid
flowchart TD
    "A[打开应用]" --> "B[查看今日任务]"
    "B" --> "C{有待完成任务?}"
    "C" --> "是" --> "D[点击打卡]"
    "D" --> "E{自定义时间?}"
    "E" --> "否" --> "F[记录当前时间]"
    "E" --> "是" --> "G[选择完成时间]"
    "F" --> "H[累加积分]"
    "G" --> "H"
    "H" --> "I[更新统计数据]"
    "I" --> "J{全部完成?}"
    "J" --> "是" --> "K[庆祝动效]"
    "J" --> "否" --> "B"
    "C" --> "否" --> "L[查看统计/兑换]"
```

### 3.2 积分兑换流程

用户进入兑换页 → 查看可用积分与兑换档位 → 点击兑换（积分不足则按钮禁用）→ 二次确认弹窗 → 扣除积分 → 生成兑换记录

```mermaid
flowchart TD
    "A[进入兑换页]" --> "B[查看积分与档位]"
    "B" --> "C{积分充足?}"
    "C" --> "是" --> "D[点击兑换]"
    "D" --> "E[二次确认弹窗]"
    "E" --> "F[确认兑换]"
    "F" --> "G[扣除积分]"
    "G" --> "H[生成兑换记录]"
    "C" --> "否" --> "I[按钮禁用+提示不足]"
```

### 3.3 每日自动刷新流程

跨天检测 → 临时任务清空 → 固定任务恢复未打卡状态 → 未完成任务不计分

## 4. 用户界面设计

### 4.1 设计风格

- **主色调**：活力橙 (#FF8C42) + 深空蓝 (#1A1B3A)，搭配暖白背景 (#FFF9F2)
- **辅助色**：成功绿 (#4ADE80)、警告红 (#EF4444)、中性灰 (#94A3B8)
- **按钮风格**：圆角胶囊按钮，主按钮渐变色，次要按钮描边
- **字体**：标题使用粗体大字号，正文使用中等字重，数字使用等宽字体突出
- **布局风格**：移动端优先，卡片式布局，底部Tab导航
- **图标风格**：使用 lucide-react 线性图标，搭配少量 emoji 增加趣味性
- **动效**：打卡成功数字滚动动画、积分变动动效、页面切换过渡

### 4.2 页面设计概览

| 页面名称 | 模块名称 | UI元素 |
|----------|----------|--------|
| 首页 | 积分概览面板 | 4个数据卡片横排，渐变背景，数字大字号，变动时滚动动画 |
| 首页 | 今日任务列表 | 未完成区（白色卡片+橙色左边框）、已完成区（浅灰卡片+绿色勾），打卡按钮为圆形 |
| 首页 | 快捷添加 | 底部浮动"+"按钮，点击弹出底部抽屉表单 |
| 任务管理 | 任务列表 | 固定任务带"每日"标签（橙色），临时任务带"临时"标签（蓝色），右侧编辑/删除图标 |
| 任务管理 | 添加表单 | 底部抽屉式弹窗，输入框+类型切换+积分输入 |
| 历史记录 | 日期筛选 | 顶部横向Tab（今日/昨日/近7天/自定义），自定义弹出日历选择器 |
| 历史记录 | 记录列表 | 按日期分组，已完成绿色标签+积分正数，未完成灰色标签+积分置灰 |
| 数据统计 | 核心面板 | 顶部4个数据卡片，与首页一致 |
| 数据统计 | 周统计图表 | CSS柱状图，7根柱子渐变色，底部星期标签，hover显示具体数值 |
| 数据统计 | 周汇总 | 3个数据卡片（周总积分/日均/最高单日） |
| 积分兑换 | 兑换档位 | 3张卡片，每张含档位名称、折扣、所需积分、兑换按钮（不足时灰色禁用） |
| 积分兑换 | 兑换记录 | 时间线样式，每条含时间、档位、扣除积分、折扣结果，顶部清空按钮 |

### 4.3 响应式设计

- 移动端优先设计，最大宽度 480px 居中
- 底部Tab导航，适配安全区域（safe-area-inset）
- 按钮最小触摸区域 44px，间距充足
- 横向滚动内容支持触摸滑动

## 5. 数据存储规则

所有数据存储在 LocalStorage，包含5个核心字段：

| 字段名 | 说明 |
|--------|------|
| tasks | 任务列表（固定+临时），含id/name/points/type/createdAt |
| checkins | 打卡记录，以日期为key，值为任务打卡详情数组 |
| stats | 积分统计数据（累计总积分等） |
| exchanges | 兑换记录数组 |
| config | 系统配置（统计起始日等） |
